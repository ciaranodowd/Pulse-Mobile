
# Core Loop

The core loop is the engine of Pulse. If this fails, the product fails.

---

## Loop Breakdown

1. User opens app
    
2. User sees venue activity
    
3. User chooses where to go
    
4. User sends a pulse
    
5. System updates data
    
6. Other users benefit
    
7. Repeat next night
    

---

## Key Drivers

- Simplicity (low friction)
    
- Instant feedback (visual + animation)
    
- Perceived accuracy of data
    

---

## Psychological Triggers

- FOMO (fear of missing out)
    
- Social proof (others are going here)
    
- Curiosity ("what’s busy tonight?")
    

---

## Critical Requirement

The loop must feel:

- Fast
    
- Accurate
    
- Alive
    

---

## Failure Points

- No data (empty app)
    
- Slow updates
    
- Confusing UI
    

---

## Optimisation Opportunities

- Push notifications to restart loop
    
- Highlight trending venues
    
- Improve animation feedback
    

---

## North Star Metric

Number of users who:

> Open Pulse before going out AND return weekly
